### Code Style & Aesthetics

#### 1. Naming
- **의도 명확화:** `temp`, `data`, `info`, `idx`, `flag` 등 모호한 범용 단어 사용 엄격히 금지
- **구체적 명시:** 변수와 함수명은 길어지더라도 그 목적과 반환값을 100% 유추할 수 있도록 구체적으로 작성
- **Boolean 타입:** 언어별 네이밍 관례를 따르되, 반드시 상태나 조동사(`is`, `has`, `can`, `should` 등)를 포함하여 참/거짓 여부를 명확히 할 것
- **약어 금지:** 누구나 아는 범용 약어(예: `id`, `url`, `html`)를 제외한 자의적인 줄임말 사용 절대 불가

#### 2. Casing & Convention
- **절대 원칙:** 파일/클래스/메서드/변수명의 대소문자 표기법(Casing)은 사용자의 기존 코드 스타일과 무관하게, **반드시 현재 작업 중인 언어의 공식 표준 스타일 가이드를 최우선으로 강제 적용**

#### 3. Formatting
- **들여쓰기:** 탭(Tab) 기호 사용 절대 금지. 스페이스(Spaces) 사용을 원칙으로 하되, 칸 수는 해당 언어의 표준 관례(예: Java 4칸, JS 2칸 등)를 따름
- **논리적 분리(Blank Lines):** 변수 선언부, 핵심 비즈니스 로직, 반환부(Return) 등 논리적 블록 사이에는 반드시 빈 줄을 1줄 추가하여 가독성 확보
